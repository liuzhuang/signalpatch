# 风险等级决定自动化边界

SignalPatch 使用四级风险分类：

- `R0`：文档、文案和非行为性测试补充；允许自动修改、合并和发布。
- `R1`：局部应用逻辑和向后兼容的缺陷修复；CI、独立审查和 Preview 验收通过后允许自动合并和发布。
- `R2`：数据库迁移、依赖升级、公开 API、认证、GitHub Workflow、Skill、Prompt 或策略变更；允许自动创建 PR 和 Preview，但合并前必须人工批准。
- `R3`：Secret、IAM、数据删除、破坏性迁移或安全边界变更；Codex 只允许分析和提交建议，不允许自动修改。

风险等级由 intake 阶段提出，由确定性策略根据变更文件和变更类型向上修正。Codex 不能自行降低风险等级。
